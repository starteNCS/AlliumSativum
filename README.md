# AlliumSativum

Federated Database Management System, 

developed as the artifact for the thesis _"Optimizing Distributed Query Execution for Heterogeneous Federated Data Systems via a Novel Cost Model and Domain-Specific Transpilation"_ by Philipp Honsel.

Please make sure to first read the thesis for a better understanding of the system, its design, and its components.
The code is organized into multiple projects, each serving a specific purpose in the overall architecture of the AlliumSativum system.

## Code overview

The tree below provides a brief overview of the different projects and their responsibilities within the AlliumSativum system.
Each project is designed to handle specific aspects of the system, from query planning and execution to worker node management and connector implementations.

```
AlliumSativum    
│
└─── AlliumSativum.QueryServer
│   │   Contains the ASP.NET Core Web API, accepting incoming query requests.
│   │   Hands off the query to AlliumSativum.QueryPlanner for planning and optimization.
│   │   Hands the resulting QExP off to AlliumSativum.QueryExecutor for execution.
│   
└─── AlliumSativum.QueryPlanner
│   │   Project contains the five steps of the query planning process as explained in Section 4.2.2 of the thesis.
│   │   Therefore, most of the optimizer's logic is implemented in this project. For the cost model refer to AlliumSativum.Shared.
│   
└─── AlliumSativum.QueryExecutor 
│   │   Contains the logic for executing the QExP generated by the QueryPlanner.
│   │   This includes all of the PlanOperatorExecutors for executing on-premises logic.
│   │   Additionally, the logic for executing the pushdown of subqueries to the data sources is implemented here.
│ 
└─── AlliumSativum.Shared
│   │   Contains shared code and utilities used across the different projects.
│   │   This includes the implementation of the cost model, as well as common data structures and helper functions.
│
└─── Workers
│   │   Contains the code for the worker nodes
│   │
│   └─── AlliumSativum.Worker
│   │   │   Contains the ASP.NET Core gRPC API for the worker nodes, allowing them to plan and execute pushdown POP's
│   │   │   Also contains the strategies for selecting the correct connector
│   │
│   └─── AlliumSativum.Worker.Proto
│   │   │   Contains the Protocol Buffers definitions for the gRPC communication between the QueryExecutor and the Worker nodes.
│   │
│   └─── AlliumSativum.Worker.Sdk
│   │   │   Contains the SDK for QueryServer -> Worker communication, abstracting the gRPC communication
│   │
│   └─── Connectors
│   │   │   Contains the code for the connectors, which are used to connect to the different data sources.
│   │   │
│   │   └─── AlliumSativum.Connectors.PostgreSql
│   │   │   Contains the connector for PostgreSQL, allowing the worker nodes to connect to PostgreSQL databases and execute queries on them.
│   │   │
│   │   └─── AlliumSativum.Connectors.JsonServer
│   │   │   Contains the connector for JSON Server, allowing the worker nodes to query JsonServer REST interfaces.

Presto
│   Contains the code for the Presto connector to connect to the JsonServer REST interface.
|   Also contains the docker-compose.yml to start up Presto.

data_analysis
│   Contains the code for the data analysis, which is used to analyze the data and generate statistics for the cost model.
│   Also contains the code for generating the visualizations of the results of the evaluation.
```
